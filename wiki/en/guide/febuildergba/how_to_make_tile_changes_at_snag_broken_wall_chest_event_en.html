I will explain how to make Tile Changes with Snag (Broken Wall), Chest, Event.
Basically, they are set using Tile Changes and Event.

Snag and Broken Wall.


Snag and Broken Wall are basically the same.
If you give damage it breaks, Tile Changes set at that point will be executed.

In the Prologue, Broken Wall can not be put because the map type is a fields.

However, if it is Snag, it can be put.
This time I will explain how to put Snag.

StepA1

Please select Map Editor.

StepA2

In Map Editor, select Snag.
It is on the top right of Map Chip.

StepA3

Please put it in a suitable place on the map.
This time it was near the river.

StepA4

Write it with “Write to ROM”.

StepA5

Next, set the Tile Changes when Snag collapses.
Please click “Add Tile Changes” button.

StepA6

As you are asked “Would you like to create additional Tile Changes?”, Select “Yes”.

StepA7

We came to the setting of “Tile Changes: 0”.
It is necessary to define Tile Changes including tiles with Snag put.

Please click “Resize” button.

StepA8

Please set X:3 Y:1 B:1.
(XY:3,1 increase 1 tile below and make it a long rectangle under 2 tiles.)

StepA9

We moved the position of Tile Changes.

StepA10

Select “Bridge”.
When Snag is defeat, it becomes Bridge.

StepA11

Fill the changing tiles with Bridge tiles.

StepA12

Write Tile Changes with “Write to ROM”.

StepA13

Start the emulator with the F5 key and let's check it.

StepA14

Attacking “Snag” …\r\n


StepA15

“Snag” was defeated, it became a bridge.

End

This is how to put Snag.
Broken Wall is also created the same way.

How to put Chest.

I will explain how to put Chest.

StepB1

Please select Map Editor.

StepB2

In Map Editor, select Chest.
It is on the top right of Map Chip.

StepB3

Please put Chest on the map.

StepB4

Write it with “Write to ROM”.

StepB5

Next, we set up Empty Chest which is the tile when Chest was opened.
Please click “Add Tile Changes” button.

StepB6

As you are asked “Would you like to create additional Tile Changes?”, Select “Yes”.

StepB8

We came to set Tile Changes: 0.
Create Empty Chest on top of Chest.

Click the “Resize” button to align the position.

StepB9

Please set X: 4 Y: 3.

StepB10

We moved the position of Tile Changes.

StepB11

Select “Empty Chest”.

StepB12

Please set “Empty Chest” in the position of Tile Changes.

StepB13

Write to ROM with “Write to ROM”.



Chest and Empty Chest are now created.
However, there are still things to set up.
It is an item in Chest.

StepB14

Let's go back to the Main Screen.
Then click on the part marked “Event”.

StepB15

We came to the “Event condition” screen.
From the “Event type” combo box, select “Map object”.



StepB16

We came to the map object setting screen.
Since there is no map object in prologue, it is empty.

StepB17

Please press the “Extend List” button to secure the place to set Chest.

StepB18

Although it will be displayed in various ways, for the time being, please do not change anything and press the “execute movement process” button.

StepB19

The map object is expanded.

However, this time, I secured five as well.
It is a waste of capacity.

When making it properly, please set the necessary capacity.

StepB20

Set the “occurrence type” to “07=Chest”.

StepB21

It switched to Chest setting.

StepB22

Set the coordinates of Chest in “Coordinate”.

StepB23

In the “contents of Chest”, set items that can be acquired.
This time, I made “ID: 88 Master Seal”.

StepB24

Please set “14=Chest” to “Type”.
(Be careful not to forget this setting.)

StepB25

Since setting is done, press “Write” button and write it to ROM.

StepB26

Chest, Empty Chest, and the contents of Chest were set.
Chest is now complete.


StepB27

Let's return to the main screen.
You can see that the Event mark is attached to the set Chest and its Chest.



I'd like to start the emulator and check the operation, but there is one problem.
That's because prologue does not have Chest Key, so you can not open Chest.
Eirika needs to have Chest Key.

StepB28

Double click on Eirika.

StepB29

Jump to the setting of Eirika of prologue.
We will have Chest Key in Eirika.

StepB30

Click item 2 on Eirika and put out item screen.

StepB31

Please select ID:69 Chest Key.
Then double click on Chest Key.

StepB32

For Eirika's belongings 2, Chest Key was set.

StepB33

Write to ROM with “write”.

StepB34

We are now ready to check.
Start the emulator with the F5 key and let's check it.

StepB35

Naturally, Eirika has a Chest Key.

StepB36

If you move Eirika over Chest, “Chest” menu will be displayed.

StepB37

We got “Master Seal”.

StepB38

Since We got the treasure, Chest becomes an empty box.

END

This concludes the explanation of Chest's setting method.

Tile Changes at Event.

I will show you how to execute Tile Changes from Event Script.


StepC1

Please select Map Editor.


StepC2

Set up changing tile.
Please click “Add Tile Changes” button.

StepC3

“Would you like to create additional Tile Changes?”と聞かれるので、“Yes”を選択してください。

StepC4

We came to set Tile Changes: 0.
Let's expand it to a tile of 6 × 6 in the meantime.

StepC5

We changed the size of Tile Changes.


StepC6

Arrange the tiles to be changed when Tile Changes happens.
Let's arrange them properly.

StepC7

Write Tile Changes with “Write to ROM”.

StepC8

Set an Event to trigger this Tile Changes.
First, return to the Main Screen.

StepC9

In the prologue, events that operate on each turn are set, so let's piggyback.
This time,Let's extend the Event that will be executed on the enemy turn of turn 1.

StepC10

Please click on the Event part.

StepC11

We came to the Event Script screen.
Double click on the “End chapter event” section at the end.

StepC12

The control panel is displayed.
Click the “Change Instruction” button.

StepC13

We came to the instruction selection screen.

StepC14

From “Category”, select “Map Relationship”.

StepC15

Because instructions related to the map are displayed, please select “Perform partial map change”.



Double click on this line or press the button to select the command.

StepC16

The instructions we chose earlier will be displayed.

StepC17

Tile Changes: Since We made Tile Changes to 0, I do not have to change anything.
Please click “New Insert” button.

StepC18

Event was inserted.

StepC19

Finally, write “Event” with the “Write” button.

StepC20

Start the emulator with the F5 key and let's check it.

StepC21

When turning back to enemy turn, Event will be executed.
(This Event is set for the Event to be executed on enemy turn of turn 1.)

End



The explanation is over with the above.
Please create more Tile Changes and enjoy.
