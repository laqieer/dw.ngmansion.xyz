Create Report Issue

This is a tool for easily reporting problems you run into so that someone more experienced may help debug your issue.

StepA1

Select “Menu” → “File” → “Create Report Issue”.

StepA 2

Please click the “Start” button.

StepA3

Please write a detailed, specific explanation. If you are vague, we may not understand you.

StepA 4

Please write your issue within the red frame, ensuring that you go into great detail on exactly how to reproduce the problem. Please also savestate shortly before the bug and/or have a save file that can easily reproduce the bug.

Please use the green highlighted box to optionally attach other files that may be relevant. Note that your save file and savestates are included automatically. For example, issues with a custom battle animation should include the assets you imported. Similarly, when problems occur with portraits, other images, or anything external that you inserted, you must include those files for us to be able to help you. Often nothing extra is needed, so it can be left blank in that case.

After entering the information, please push the next button.

StepA 5

FEBuilderGBA automatically looks for the unmodified ROM from the current directory, but if it cannot find your vanilla ROM, then you must specify the filepath to it.

Please press the create button to start creating the report.7z.

StepA 6

Please wait as it processes, as it takes a little while.

FEBuilderGBA creates a .ups patch of your current ROM along with patches of a few of your backups that share the same base ROM name. It then compresses it into a 7z package.

StepA 7

A report.7z file was created on your computer. Please share it to feuniverse.us or via discord so that we may help.

Please note that there is no illegal ROM in this file - only .ups is stored.
In addition, your computer's username is protected as all instances of filepaths in the log are edited. For example, the path c:\users\who\ replaces %USER% in the febuilder debugging log.

You can ask for assistance via the Discord link below. Click here for details.

commentary on report.7z

I will explain the information stored in report.7z.

A

It is ups data of the current ROM.
This is the target to fix.

B

It is a save file of the emulator.
If save files of VBA and mGBA exist, they are stored.

C

If you make all backups ups, it takes a lot of time, so we have backed up samples of the following generations. 

int[] olderPickup = new int [] {0, 1, 2, 3, 4, 5, 10, 15, 20, 30, 40, 60, 80, 100, 140, 180, 250, 300, 400, 600, 800, 1000};

D

Paths such as c:\users\who\ in the log are converted to %USER%.
At the end of the log, the symptoms of the bug, the version of FEBuilderGBA, the version of the game etc. are listed.

E

If you have the attached file, it will be stored here.

F

The etc directory stores the names of comments and flags set during development.

A lot of files are stored, but due to the high compression of 7z, the file size will be smaller than one uncompressed ups.

FAQ

Q1

Why do I need to create report7z?

Answer:
Because it is to reliably reproduce the your problem and restore it.
It is the same as a medical doctor examining a patient.
Medical doctor can not treat without examining them.

report7z contains all the information necessary to reproduce your problem.
Therefore, we can accurately reproduce your problem and fix it.

Q2

Why do I need report7z instead of bulletin board exchange?

Answer:
Because It is to shorten the time.
We live in different time zones.
Therefore, it takes time to check the parameters on the bulletin board.
Also, we can not give accurate advice unless we can confirm your status.
To solve this problem at once, report7z was created.

Q3

Does report7z include ROM?

Answer:
No.
Illegal ROM is not included at all.
Report7z contains only ups patches.

Q4

Does report7z include my personal information?

Answer:
No.
Because the mask processing is executed in the log included in report7z, your personal information is not included.
The path of FEBuilderGBA is replaced by %BASEDIR%.
The user's home directory is replaced with %USER%.

Conversion example
c:\users\foo\Desktop\FEBuilderGBA\roms\target.gba
↓
%BASEDIR%/roms/target.gba

c:\users\foo\roms\target.gba
↓
%USER%/roms/target.gba
